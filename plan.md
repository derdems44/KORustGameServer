## Test Strategy

### Unit Tests (per handler)
- Packet roundtrip (serialize → deserialize)
- Validation logic (requirements, limits)
- Formula verification (damage, HP, XP)

### Integration Tests
- Full login → char select → game start → move → zone change
- Combat: attack → damage → death → respawn
- Trade: request → add items → confirm → verify inventories
- Party: create → invite → accept → XP share

### Client Compatibility Tests
1. Single client: login, enter game, walk around, see NPCs
2. Two clients: see each other, chat, trade, party
3. Combat test: attack NPC, use skill, die, respawn
4. Zone change: walk through warp gate, verify new zone

### Load Tests
- 100 concurrent connections
- NPC AI with 1000+ active monsters per zone
- Region broadcast performance
